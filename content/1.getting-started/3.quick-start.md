# Quick Start

Build your first type-safe REST client in minutes.

## Créer votre premier modèle

### Définir un modèle avec des décorateurs

Create a `models/User.ts` file:

```ts [models/User.ts]
import {
  Model,
  Resource,
  Field,
  Key,
} from "laravel-raom-nuxt/runtime";

@Resource("users", { limits: [10, 25, 50] })
class User extends Model {
  @Key()
  @Field()
  id!: number;

  @Field()
  firstname!: string;

  @Field()
  lastname!: string;

  @Field()
  email!: string;

  @Field()
  active: boolean = true;

  // Computed property
  get fullName(): string {
    return `${this.firstname} ${this.lastname}`;
  }
}

export default User;
```

**Key points:**

- `@Resource('users')` — maps to `/users` API endpoint
- `@Key() @Field() id` — primary key field
- `@Field()` — marks fields for hydration and serialization
- Computed properties (no `@Field`) are not persisted

## Effectuer votre première requête

Import your model and query it:

```ts [pages/index.vue]
<script setup lang="ts">
import User from '~/models/User'

// Find by primary key
const user = await User.query().findByKey(1)
console.log(user.fullName)

// Get all users
const [users] = await User.query().get()

// Filter and sort
const [activeUsers] = await User.query()
  .where('active', '=', true)
  .orderBy('lastname', 'asc')
  .get()
</script>

<template>
  <div>
    <h1>{{ user.fullName }}</h1>
    <p>{{ user.email }}</p>
  </div>
</template>
```

## Ajouter des relations

Define related models with relation builders:

```ts [models/User.ts]
import {
  Model,
  Resource,
  Field,
  Key,
  HasMany,
  HasOne,
} from "laravel-raom-nuxt/runtime";
import Post from "./Post";
import Profile from "./Profile";

@Resource("users")
class User extends Model {
  @Key() @Field() id!: number;
  @Field() firstname!: string;
  @Field() lastname!: string;

  posts = HasMany(() => Post, "posts");
  profile = HasOne(() => Profile, "profile");
}

export default User;
```

```ts [models/Post.ts]
import {
  Model,
  Resource,
  Field,
  Key,
  BelongsTo,
} from "laravel-raom-nuxt/runtime";
import User from "./User";

@Resource("posts")
class Post extends Model {
  @Key() @Field() id!: number;
  @Field() title!: string;
  @Field() content!: string;
  @Field() published: boolean = false;
  @Field() userId!: number;

  author = BelongsTo(() => User, "author");
}

export default Post;
```

**Note:** Use arrow functions `() => Post` to avoid circular dependency issues.

## Requêtes avec relations

Eager-load relations with `include()`:

```ts
// Load user with posts
const user = await User.query().include("posts").findByKey(1);

const posts = user.posts.get();
console.log(posts.length);

// Load with constraints
const user = await User.query()
  .include("posts", (q) =>
    q.where("published", "=", true).orderBy("created_at", "desc").limit(5),
  )
  .findByKey(1);

// Nested includes
const user = await User.query()
  .include("posts", (q) => q.include("comments"))
  .include("profile")
  .findByKey(1);
```

## Mutations

### Modifier un modèle

Field changes are tracked automatically:

```ts
const user = await User.query().findByKey(1);

// Mutate fields
user.firstname = "Jane";
user.lastname = "Doe";

// Check pending changes
console.log(user._changes); // { firstname: 'Jane', lastname: 'Doe' }

// Commit to API
await user.save();

// Changes cleared after successful save
console.log(user._changes); // {}
```

### Mutations de relations

Relation mutations are deferred until `save()`:

```ts
const user = await User.query().include("posts", "roles").findByKey(1);
const newPost = await Post.query().findByKey(10);

// Queue relation mutations
user.posts.attach(newPost);
user.posts.detach(3);
user.posts.create({ title: "New Post", content: "Hello!" });

user.roles.sync([1, 2, 3]);
user.roles.toggle(4);

// All mutations committed atomically
await user.save();
```

**Available operations:**

- `attach(id | instance)` — add to relation
- `detach(id | instance)` — remove from relation
- `sync(ids)` — replace with specific set
- `toggle(id | instance)` — add if absent, remove if present
- `create(attributes)` — create new related instance

### Exemple complet

```ts
<script setup lang="ts">
import User from '~/models/User'
import Post from '~/models/Post'

// Load user
const user = await User.query().include('posts').findByKey(1)

// Mutate fields
user.firstname = 'Jane'

// Create new post
user.posts.create({
  title: 'My First Post',
  content: 'This is my first post!',
  published: true
})

// Detach old posts
user.posts.detach([5, 7])

// Save all changes
try {
  await user.save()
  console.log('Saved successfully!')
} catch (error) {
  console.error('Save failed:', error)
  // _changes preserved on error — can retry
}
</script>
```
