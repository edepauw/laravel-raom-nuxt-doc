# Dirty Tracking

Proxy-based change tracking that captures field mutations and defers them until `save()`.

## Qu'est-ce que le Dirty Tracking ?

Dirty tracking is the mechanism that intercepts field assignments on a model and records them as pending changes. Changes are stored separately from the committed ("clean") field values, so you always know what has changed since the last save.

## Comment ça fonctionne ?

### Proxy-based tracking

Every model instance is wrapped in a JavaScript `Proxy`. When you assign a value to a field decorated with `@Field()`, the proxy intercepts the assignment and stores it in `_changes` instead of applying it directly.

### _fields et _changes

Two internal objects track state:

- **`_fields`** — committed field values (what was last saved/loaded from the API)
- **`_changes`** — pending mutations (what will be sent on the next `save()`)

```ts
const user = await User.query().findByKey(1)

// Initial state after hydration
console.log(user._fields)  // { id: 1, firstname: 'John', lastname: 'Doe', ... }
console.log(user._changes) // {}

// Mutation — stored in _changes, not _fields
user.firstname = 'Jane'

console.log(user._fields)  // { id: 1, firstname: 'John', ... } (unchanged)
console.log(user._changes) // { firstname: 'Jane' }
```

When reading a field, `_changes` takes priority over `_fields`, so `user.firstname` returns `'Jane'` immediately after the assignment.

## Détecter les modifications

### Vérifier si un modèle a été modifié

```ts
if (user.isDirty()) {
  // The model has pending changes
}
```

`isDirty()` returns `true` if `_changes` is non-empty **or** if the model is new (`_isNew = true`).

### Vérifier quels champs ont été modifiés

Inspect `_changes` directly:

```ts
const pendingChanges = user._changes
// e.g. { firstname: 'Jane', lastname: 'Doe' }
```

### Obtenir la valeur originale d'un champ

Read from `_fields` to get the committed value:

```ts
const originalFirstname = user._fields['firstname']
// Returns the value as loaded from the API, before any pending changes
```

## Sauvegarder les modifications

### La méthode save()

Call `save()` to commit pending changes to the API:

```ts
user.firstname = 'Jane'
await user.save()
```

Internally, `save()`:

1. Builds a mutation payload from `_changes` (fields) and any queued relation operations
2. Sends a `POST` request to the API
3. On success, calls `applyChanges()` to promote `_changes` into `_fields` and clear the pending changes

### Commit des modifications

After a successful `save()`, all pending changes are committed:

```ts
user.firstname = 'Jane'
console.log(user._changes) // { firstname: 'Jane' }

await user.save()
console.log(user._changes) // {} — cleared after success
console.log(user._fields.firstname) // 'Jane' — now committed
```

### Gestion des erreurs

If `save()` throws, `_changes` are **preserved** so you can inspect them, adjust, and retry:

```ts
user.firstname = 'Jane'
try {
  await user.save()
} catch (error) {
  console.log(user._changes) // { firstname: 'Jane' } — still pending
  // You can inspect the error, fix the data, and retry
}
```

## Annuler les modifications

### Annuler tous les changements

Use `discardChanges()` to drop all pending field changes:

```ts
user.firstname = 'Jane'
console.log(user._changes) // { firstname: 'Jane' }

user.discardChanges()
console.log(user._changes) // {}
console.log(user.firstname) // 'John' — reverted to _fields value
```

**Note:** `discardChanges()` only clears `_changes`. It does not affect queued relation mutations.

### Annuler un champ spécifique

To revert a single field, delete it from `_changes`:

```ts
user.firstname = 'Jane'
user.lastname = 'Smith'

delete user._changes['firstname']
// firstname reverts to _fields value; lastname still pending
```

## Cas d'usage avancés

### Validation avant sauvegarde

```ts
user.firstname = 'Jane'

if (user.isDirty() && user.firstname.length > 0) {
  await user.save()
} else {
  user.discardChanges()
}
```

### Optimistic updates

```ts
const user = await User.query().findByKey(1)

// Optimistic update — UI reflects change immediately
const originalFirstname = user._fields['firstname'] as string
user.firstname = 'Jane'

try {
  await user.save()
} catch (error) {
  // Rollback on failure — discard pending change and restore the original committed value
  user.discardChanges()
  user.setField('firstname', originalFirstname, false) // false = write to _fields without tracking as a change
}
```
