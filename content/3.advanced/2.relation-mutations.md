# Mutate

Dirty tracking and deferred relation mutations let you batch changes and commit them atomically.

## Mutations différées

### Principe des mutations différées

Field changes and relation mutations are tracked in memory and committed only when you call `save()`:

```ts
user.firstname = "Jane"; // tracked in _changes
user.posts.attach(post); // queued for commit
user.roles.sync([1, 2, 3]); // queued for commit

await user.save(); // send all mutations in one request
```

**Benefits:**

- ✅ Atomic commits — all changes succeed or all fail
- ✅ Automatic rollback on error
- ✅ Batch multiple changes efficiently
- ✅ Visible tracking of pending changes

### Dirty tracking for fields

Field mutations are tracked in `_changes`:

```ts
user.firstname = "Jane";
user.lastname = "Doe";

console.log(user._changes); // { firstname: 'Jane', lastname: 'Doe' }

await user.save(); // commits _changes to API

console.log(user._changes); // {} (cleared on success)
```

If `save()` fails, `_changes` are preserved and can be inspected or retried.

## Opérations sur les relations

Relation mutations are deferred until `save()`. Available operations:

- **`attach()`** — add related instances
- **`detach()`** — remove related instances
- **`sync()`** — replace with a specific set
- **`toggle()`** — add if absent, remove if present
- **`create()`** — create new related instances

## Attach

Add related instances to a relation.

### Attacher un élément

```ts
const post = await Post.query().findByKey(5);
user.posts.attach(post);
await user.save();
```

### Attacher par ID

```ts
user.posts.attach(5); // attach post with id=5
await user.save();
```

### Attacher plusieurs éléments

```ts
user.posts.attach([post1, post2, post3]);
await user.save();
```

Or with IDs:

```ts
user.posts.attach([5, 7, 9]);
await user.save();
```

### Attacher avec données pivot

For `BelongsToMany` relations, you can attach pivot attributes:

```ts
user.roles.attach(role, { expires_at: "2024-12-31", level: "admin" });
await user.save();
```

Pivot data is stored in the join table.

## Detach

Remove related instances from a relation.

### Détacher un élément

```ts
user.posts.detach(post);
await user.save();
```

### Détacher par ID

```ts
user.posts.detach(3);
await user.save();
```

### Détacher plusieurs éléments

```ts
user.posts.detach([1, 2, 3]);
await user.save();
```

### Détacher tous les éléments

```ts
user.posts.detach(); // detach all
await user.save();
```

## Sync

Replace the entire relation with a specific set of IDs.

### Synchroniser avec un ensemble d'IDs

```ts
user.roles.sync([1, 2, 3]);
await user.save();
```

After `save()`, the user will have **only** roles 1, 2, and 3. All other roles are detached.

### Sync sans détacher

Add IDs without removing existing ones:

```ts
user.roles.sync([4, 5], { withoutDetaching: true });
await user.save();
```

Now roles 4 and 5 are added, but existing roles remain.

### Sync avec données pivot

```ts
user.roles.sync([
  { id: 1, pivot: { expires_at: "2025-01-01" } },
  { id: 2, pivot: { expires_at: "2025-06-01" } },
]);
await user.save();
```

## Toggle

Add if absent, remove if present.

### Basculer la présence d'un élément

```ts
user.roles.toggle(role);
await user.save();
```

If `role` is already attached, it's detached. If not, it's attached.

### Toggle par ID

```ts
user.roles.toggle(3);
await user.save();
```

### Toggle multiple

```ts
user.roles.toggle([1, 2, 3]);
await user.save();
```

Each ID is toggled independently.

## Create

Create new related instances.

### Créer un élément lié

```ts
user.posts.create({ title: "New Post", content: "Hello world!" });
await user.save();
```

Creates a new post associated with the user.

### Créer plusieurs éléments

```ts
user.posts.create([
  { title: "Post 1", content: "First" },
  { title: "Post 2", content: "Second" },
]);
await user.save();
```

### Create avec données pivot

For `BelongsToMany`:

```ts
user.roles.create({ name: "Editor" }, { expires_at: "2025-12-31" });
await user.save();
```

The second argument is pivot data.

## Mutations imbriquées

### Relations récursives

You can nest mutations deeply:

```ts
user.posts.create({
  title: "New Post",
  content: "Hello!",
  comments: {
    create: [
      { content: "First comment", author_id: 2 },
      { content: "Second comment", author_id: 3 },
    ],
  },
});

await user.save();
```

This creates:

1. A new post for the user
2. Two comments for the new post

All in one `save()` call.

## Exemple complet

```ts
const user = await User.query().include("posts", "roles").findByKey(1);

// Field mutations
user.firstname = "Jane";
user.lastname = "Doe";

// Relation mutations
user.posts.attach([5, 7]);
user.posts.detach(3);
user.posts.create({ title: "New Post", content: "Hello!" });

user.roles.sync([1, 2, 3]);
user.roles.toggle(4);

// All mutations committed atomically
await user.save();
```

If any mutation fails, all changes are rolled back.

## Gestion du payload

### Construction du payload

The mutation payload is built automatically from `_changes` and queued relation mutations:

```ts
{
  "id": 1,
  "firstname": "Jane",
  "lastname": "Doe",
  "posts": {
    "attach": [5, 7],
    "detach": [3],
    "create": [{ "title": "New Post", "content": "Hello!" }]
  },
  "roles": {
    "sync": [1, 2, 3],
    "toggle": [4]
  }
}
```

### Validation des mutations

Mutations are validated by the Laravel backend. If validation fails, `save()` throws and `_changes` are preserved.

## Bonnes pratiques

### Always await save()

```ts
// ✅ Correct
user.firstname = "Jane";
await user.save();

// ❌ Forgot to save — changes never committed
user.firstname = "Jane";
```

### Batch multiple changes

```ts
// ✅ Efficient — one request
user.firstname = "Jane";
user.lastname = "Doe";
user.posts.attach([5, 7]);
await user.save();

// ❌ Inefficient — multiple requests
user.firstname = "Jane";
await user.save();
user.lastname = "Doe";
await user.save();
user.posts.attach([5, 7]);
await user.save();
```

### Handle errors gracefully

```ts
try {
  user.firstname = "Jane";
  user.posts.attach(5);
  await user.save();
} catch (error) {
  console.error("Save failed:", error);
  // _changes are preserved — can inspect or retry
  console.log(user._changes);
}
```

### Use sync() for complete replacement

```ts
// ✅ Replace roles entirely
user.roles.sync([1, 2, 3]);

// ❌ Detach all then attach — same result but less efficient
user.roles.detach();
user.roles.attach([1, 2, 3]);
```

### Inspect pending changes

```ts
user.firstname = "Jane";
user.posts.attach(5);

console.log(user._changes); // { firstname: 'Jane' }
console.log(user.posts._mutations); // internal mutation queue

await user.save();

console.log(user._changes); // {} (cleared after successful save)
```
