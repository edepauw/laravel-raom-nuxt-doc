# Contributing

Contributions are welcome! This guide will help you get started.

## Comment contribuer ?

::callout{type="success"}
The project is open source and maintained by **edepauw**. Contributions, bug reports, and feature requests are all appreciated!
::

Ways to contribute:

- 🐛 Report bugs and issues
- 💡 Suggest new features
- 📝 Improve documentation
- 🧪 Write tests
- 💻 Submit pull requests

## Domaines de contribution prioritaires

These areas have the highest impact and are actively seeking contributors:

### Fake engine + factories

Implement the fake query mode and factory system:

- In-memory data storage
- `User.fake().findByKey(1)` API
- Factory definitions and data generation
- Faker integration
- Seeding utilities

**Impact:** Enables frontend development without a backend, fast tests.

### Support des relations Morph

Add polymorphic relation support:

- `morphTo()`, `morphMany()`, `morphOne()` builders
- Hydration and mutation for morph relations
- Integration with identity map

**Impact:** Unlocks flexible data modeling (e.g., comments on multiple parent types).

### Lifecycle hooks

Implement model lifecycle events:

- `beforeSave`, `afterSave`
- `beforeCreate`, `afterCreate`
- `beforeUpdate`, `afterUpdate`
- `beforeDelete`, `afterDelete`
- `afterHydrate`

**Impact:** Enables validation, transformation, and side effects at key lifecycle points.

### Tests d'intégration

Create integration tests with a real Laravel backend:

- Spin up Laravel API in CI (Docker)
- Test queries, mutations, and error handling end-to-end
- Validate compatibility with [`lomkit/laravel-rest-api`](https://github.com/lomkit/laravel-rest-api)

**Impact:** Ensures compatibility and catches regressions.

## Configuration de l'environnement de développement

### Installation

Clone the repository and install dependencies:

```bash
# Clone the repo
git clone https://github.com/your-username/laravel-raom-nuxt.git
cd laravel-raom-nuxt

# Install dependencies
npm install

# Prepare module stubs
npm run dev:prepare
```

### Développement

Run the playground for local development:

```bash
# Start playground in dev mode
npm run dev

# Build playground
npm run dev:build
```

The playground is a Nuxt app located in `playground/` where you can test changes.

### Tests

Run the test suite:

```bash
# Run all tests
npm run test

# Watch mode for TDD
npm run test:watch

# Lint
npm run lint

# Type check
npm run typecheck
```

**Test coverage:** Aim for 80%+ coverage on new code.

### Conventions de code

- **TypeScript** — all code is TypeScript
- **ESLint** — lint with `npm run lint`
- **Prettier** — format on save (config in `.prettierrc`)
- **Conventional Commits** — commit messages follow [Conventional Commits](https://www.conventionalcommits.org/)

**Example commits:**

- `feat: add morphTo relation support`
- `fix: identity map not clearing on detach`
- `docs: update query builder examples`

## Soumettre une contribution

### 1. Fork et clone

Fork the repository on GitHub, then clone your fork:

```bash
git clone https://github.com/your-username/laravel-raom-nuxt.git
cd laravel-raom-nuxt
```

Add the upstream repository:

```bash
git remote add upstream https://github.com/edepauw/laravel-raom-nuxt.git
```

### 2. Créer une branche

Create a feature branch:

```bash
git checkout -b feature/my-feature
```

**Naming conventions:**

- `feature/` — new features
- `fix/` — bug fixes
- `docs/` — documentation updates
- `test/` — test additions

### 3. Développer et tester

Make your changes, then test:

```bash
# Run tests
npm run test

# Run lint
npm run lint

# Type check
npm run typecheck
```

### 4. Commit

Commit with a conventional commit message:

```bash
git add .
git commit -m "feat: add support for morph relations"
```

**Commit message format:**

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Types:** `feat`, `fix`, `docs`, `test`, `refactor`, `chore`

### 5. Pull Request

Push your branch:

```bash
git push origin feature/my-feature
```

Open a pull request on GitHub. Include:

- **Description** — what does this PR do?
- **Motivation** — why is this change needed?
- **Testing** — how did you test this?
- **Breaking changes** — any breaking changes?

## Reporting de bugs

Found a bug? Please report it!

### Informations à fournir

When reporting a bug, include:

1. **Description** — clear summary of the issue
2. **Steps to reproduce** — minimal reproducible example
3. **Expected behavior** — what should happen?
4. **Actual behavior** — what actually happens?
5. **Environment** — Node version, Nuxt version, module version
6. **Code sample** — ideally a GitHub repo or CodeSandbox

**Example:**

```markdown
## Bug Report

### Description

Identity map not clearing after `detach()` operation.

### Steps to Reproduce

1. Load user with posts: `User.query().include('posts').findByKey(1)`
2. Detach a post: `user.posts.detach(5)`
3. Save: `await user.save()`
4. Check identity map — post 5 still cached

### Expected Behavior

Post 5 should be removed from identity map.

### Actual Behavior

Post 5 remains in identity map.

### Environment

- Node: 20.10.0
- Nuxt: 3.10.0
- Module: 0.1.0
```
