# Roadmap

The module is under active development. This roadmap outlines completed milestones and planned features.

## État actuel

::callout{type="info"}
The core architecture is implemented and covered by an automated test suite (88 tests passing). The module is **not yet production-ready** but actively moving toward stability.
::

**Current status:**

- ✅ Decorator metadata system (`@Resource`, `@Field`, `@Key`)
- ✅ Identity Map + instance reuse
- ✅ Proxy-based dirty tracking
- ✅ `save()` with deferred commit
- ✅ Relation builders (`HasMany`, `BelongsTo`, `HasOne`, `BelongsToMany`)
- ✅ Relation operations (`attach`, `detach`, `sync`, `toggle`, `create`)
- ✅ Nested/recursive relation mutation payloads
- ✅ Query Builder (filters, sorts, includes, pagination)
- 🚧 Fake engine (planned)
- 🚧 Morph relations (planned)
- 🚧 Lifecycle hooks (planned)

## Phase 4 - Fake Engine 🚧

**Status:** Planned

Enable frontend development without a live backend.

### fake() et FakeQueryBuilder

Identical API, zero network calls:

```ts
// Real query — hits API
const user = await User.query().findByKey(1);

// Fake query — in-memory
const user = await User.fake().findByKey(1);
```

**Benefits:**

- ⚡ Fast, deterministic tests
- 🧪 Frontend dev without backend
- 🏭 Factory-based data generation

### Factories / Seeders en mémoire

Define factories for generating test data:

```ts
UserFactory.define({
  firstname: () => faker.name.firstName(),
  lastname: () => faker.name.lastName(),
  email: () => faker.internet.email(),
});

const users = UserFactory.createMany(10);
```

Seed in-memory database for fake mode.

### Garantie de parité d'API

Ensure real and fake modes have identical APIs. Tests should run against both without changes.

## Idées de fonctionnalités futures 💡

Features under consideration for future phases.

### Offline-first mutation queue

Queue mutations when offline, replay when back online:

- [ ] Persistent mutation queue (IndexedDB)
- [ ] Automatic retry on reconnect
- [ ] Conflict resolution for concurrent edits

**Use case:** Mobile apps, unreliable networks.

### Lifecycle events

Hooks for model lifecycle:

- `beforeSave` — validate before commit
- `afterHydrate` — transform data after loading
- `afterDelete` — cleanup after deletion
- `beforeCreate`, `afterCreate`
- `beforeUpdate`, `afterUpdate`

**Use case:** Validation, transformation, side effects.

### Policy/gate helpers

Local pre-check for Laravel policies:

```ts
if (user.can("update", post)) {
  // Show edit button
}
```

**Note:** Not a security mechanism — always validate on the backend.

### Normalisation des erreurs Laravel

Transform Laravel validation errors into structured objects:

```ts
try {
  await user.save();
} catch (error) {
  if (error.isValidationError()) {
    console.log(error.fields.email); // ['Email already taken']
  }
}
```

**Goal:** Easier error handling, better UX.

### Génération de code

Generate TypeScript models from Laravel API schema:

```bash
npx laravel-rest-api-mapper generate
```

Auto-generates models with fields, relations, and types.

**Goal:** Eliminate boilerplate, stay in sync with backend.
