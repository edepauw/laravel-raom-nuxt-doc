# Models & Decorators

Define your domain models with TypeScript decorators. Metadata is collected at runtime to drive hydration, serialization, and API communication.

## La classe Model

All models extend the `Model` base class:

```ts
import {
  Model,
  Resource,
  Field,
  Key,
} from "laravel-rest-api-object-mapper/runtime";

@Resource("users")
class User extends Model {
  @Key() @Field() id!: number;
  @Field() firstname!: string;
  @Field() lastname!: string;
}
```

The `Model` class provides:

- **Query builder** — `User.query()`
- **Identity map integration** — automatic instance reuse
- **Dirty tracking** — field changes tracked in `_changes`
- **Lifecycle methods** — `save()`, `delete()`, `refresh()`

## Le décorateur @Resource

Configures the REST resource endpoint and options.

### Syntaxe de base

```ts
@Resource("users")
class User extends Model {
  // ...
}
```

This maps the `User` model to the `/users` API endpoint.

### Options de configuration

```ts
@Resource("users", { limits: [1, 10, 25, 50] })
class User extends Model {
  // ...
}
```

Available options:

- **`limits`** — available page sizes for pagination
- **`defaultLimit`** — default page size (defaults to first limit)

## Le décorateur @Field

Marks properties that should be hydrated from API responses and serialized for API requests.

### Déclarer des champs

```ts
@Field()
firstname!: string

@Field()
age!: number

@Field()
active!: boolean
```

Only properties marked with `@Field()` are:

- ✅ Hydrated from API responses
- ✅ Included in mutation payloads
- ✅ Tracked for dirty checking

Properties without `@Field()` are ignored by the mapper.

### Types de champs supportés

All JSON-serializable types:

```ts
@Field() name!: string
@Field() age!: number
@Field() active!: boolean
@Field() tags!: string[]
@Field() metadata!: Record<string, any>
@Field() createdAt!: Date  // serialized as ISO string
```

**Note:** Dates are serialized as ISO strings. You can use computed getters for custom formatting.

## Le décorateur @Key

Identifies primary key fields. Required for identity map and `findByKey()`.

### Clés primaires

```ts
@Key()
@Field()
id!: number
```

The `@Key()` decorator must be combined with `@Field()`.

**Important:** Every model must have at least one `@Key()` field.

### Clés composées

For composite keys, use multiple `@Key()` decorators:

```ts
@Key() @Field() userId!: number
@Key() @Field() roleId!: number
```

The identity map will use the combination of all key values.

## Exemple complet

```ts
import {
  Model,
  Resource,
  Field,
  Key,
  HasMany,
  BelongsTo,
} from "laravel-rest-api-object-mapper/runtime";

@Resource("users", { limits: [10, 25, 50, 100] })
class User extends Model {
  @Key()
  @Field()
  id!: number;

  @Field()
  firstname!: string;

  @Field()
  lastname!: string;

  @Field()
  email!: string;

  @Field()
  age?: number;

  @Field()
  active: boolean = true;

  @Field()
  createdAt!: Date;

  // Computed property (not a field)
  get fullName(): string {
    return `${this.firstname} ${this.lastname}`;
  }

  // Relations
  posts = HasMany(() => Post, "posts");
  profile = HasOne(() => Profile, "profile");
}
```

## Bonnes pratiques

### Use definite assignment for required fields

```ts
@Field()
firstname!: string  // ✅ Required field
```

### Use optional type for nullable fields

```ts
@Field()
middlename?: string  // ✅ Optional field
```

### Provide defaults for boolean fields

```ts
@Field()
active: boolean = true  // ✅ Default value
```

### Don't mark relations with @Field

```ts
// ❌ Wrong
@Field()
posts = HasMany(() => Post, 'posts')

// ✅ Correct
posts = HasMany(() => Post, 'posts')
```

Relations self-register their metadata — no `@Field` or `@Relation` decorator needed.

### Use computed properties for derived data

```ts
get fullName(): string {
  return `${this.firstname} ${this.lastname}`
}
```

Computed properties are not fields — they won't be serialized or hydrated.
