# Query Builder

Chain filters, sorts, includes, and pagination with a fluent, type-safe API.

## Introduction au Query Builder

The query builder provides a fluent interface for constructing API requests:

```ts
const [users] = await User.query()
  .where("firstname", "like", "Eli%")
  .where("age", ">", 18)
  .orderBy("lastname", "asc")
  .include("posts")
  .limit(25)
  .get();
```

Every model has a static `query()` method that returns a new query builder instance.

## Requêtes de base

### Récupérer par clé primaire

```ts
const user = await User.query().findByKey(1);
```

Returns a single model instance or throws if not found.

For composite keys:

```ts
const pivot = await UserRole.query().findByKey({ userId: 1, roleId: 5 });
```

### Récupérer plusieurs ressources

```ts
const [users] = await User.query().get();
```

Returns a tuple `[data, meta]` where `data` is an array of model instances and `meta` contains pagination metadata.

### Récupérer la première ressource

```ts
const user = await User.query().where("email", "=", "john@example.com").first();
```

Returns the first matching instance or `null`. Requires `1` to be in the resource's `limits` array.

> **Note:** `first()` calls `limit(1)` internally. Make sure your `@Resource` decorator includes `1` in its `limits` option: `@Resource('users', { limits: [1, 10, 25, 50] })`.

## Filtres

### Clause where

```ts
const [users] = await User.query().where("firstname", "like", "John%").get();
```

**Syntax:** `where(field, operator, value)`

### Filtres multiples

Chain multiple `where()` calls:

```ts
const [users] = await User.query()
  .where("age", ">", 18)
  .where("active", "=", true)
  .where("country", "=", "FR")
  .get();
```

All conditions are combined with `AND`.

### Filtre OR

Use `orWhere()` for OR conditions:

```ts
const [users] = await User.query()
  .where("role", "=", "admin")
  .orWhere("role", "=", "moderator")
  .get();
```

### Opérateurs supportés

- **`=`** — equal
- **`!=`** — not equal
- **`>`** — greater than
- **`>=`** — greater than or equal
- **`<`** — less than
- **`<=`** — less than or equal
- **`like`** — pattern matching (use `%` wildcards)
- **`not like`** — inverse pattern matching
- **`in`** — value in array
- **`not in`** — value not in array
- **`between`** — value between two bounds
- **`not between`** — value outside two bounds

Examples:

```ts
// Like
User.query().where("email", "like", "%@example.com");

// In
User.query().where("status", "in", ["active", "pending"]);

// Not in
User.query().where("role", "not in", ["admin", "moderator"]);
```

## Tri

### Tri simple

```ts
const [users] = await User.query().orderBy("firstname", "asc").get();
```

**Syntax:** `orderBy(field, direction)`  
**Direction:** `'asc'` or `'desc'`

### Tri multiple

```ts
const [users] = await User.query()
  .orderBy("lastname", "asc")
  .orderBy("firstname", "asc")
  .get();
```

Sorts are applied in the order they're chained.

## Recherche full-text

Use `text()` to perform a full-text search:

```ts
const [users] = await User.query().text("John").get();
```

## Scopes Laravel

Apply Laravel named scopes with `scope()`:

```ts
const [users] = await User.query().scope("active").get();

// With parameters
const [users] = await User.query().scope("olderThan", 18).get();
```

## Agrégations

Compute aggregates on relations with `aggregate()` or its shorthand methods:

```ts
const [users] = await User.query()
  .withCount("posts")          // adds posts_count
  .withSum("posts", "views")   // adds posts_sum_views
  .withAvg("posts", "rating")  // adds posts_avg_rating
  .withMin("posts", "created_at")
  .withMax("posts", "created_at")
  .get();
```

## Inclusion de relations

### Include simple

```ts
const user = await User.query().include("posts").findByKey(1);
```

Eager-loads the `posts` relation.

### Include avec contraintes

Apply filters and sorts to the included relation:

```ts
const user = await User.query()
  .include("posts", (q) => q.orderBy("created_at", "desc").limit(5))
  .findByKey(1);
```

The callback receives a query builder for the related model.

### Include imbriqués

Nest includes for deep loading:

```ts
const user = await User.query()
  .include("posts", (q) => q.include("comments", (c) => c.include("author")))
  .findByKey(1);
```

Loads users → posts → comments → authors in one request.

### Include multiples

Chain multiple `include()` calls:

```ts
const user = await User.query()
  .include("posts")
  .include("profile")
  .include("roles")
  .findByKey(1);
```

## Pagination

### Pagination basique

```ts
const [users] = await User.query().limit(10).page(1).get();
```

- **`limit(n)`** — number of results per page (must be in the resource's `limits` array)
- **`page(n)`** — page number (1-based)

### Pagination avec métadonnées

`get()` returns a tuple `[data, meta]`. The `meta` object contains pagination information from the Laravel backend:

```ts
const [users, meta] = await User.query().limit(25).page(1).get();

console.log(users);         // model instances
console.log(meta.total);    // total count
console.log(meta.per_page); // items per page
console.log(meta.current_page);
```

### Récupérer une page spécifique

Use `getPage()` as a shorthand:

```ts
const [users, meta] = await User.query().limit(25).getPage(2);
```

## Sélection de champs

Limit which fields are returned:

```ts
const [users] = await User.query().select("id", "firstname", "lastname").get();
```

**Note:** `select()` takes individual field names as rest parameters (not an array). Primary key fields are always included.

Useful for reducing payload size when you only need specific fields.

## Exemple complet

Combining all features:

```ts
const [users, meta] = await User.query()
  .where("age", ">=", 18)
  .where("active", "=", true)
  .where("country", "in", ["FR", "BE", "CH"])
  .orderBy("lastname", "asc")
  .orderBy("firstname", "asc")
  .include("posts", (q) =>
    q.where("published", "=", true).orderBy("created_at", "desc").limit(5),
  )
  .include("profile")
  .select("id", "firstname", "lastname", "email")
  .limit(25)
  .page(1)
  .get();
```

This query:

1. Filters users by age, active status, and country
2. Sorts by lastname then firstname
3. Includes published posts (latest 5) and profile
4. Selects specific fields
5. Paginates (25 per page, first page)

## Bonnes pratiques

### Always use typed field names

```ts
// ✅ Type-safe
User.query().where("firstname", "like", "John%");

// ❌ Typo will cause runtime error
User.query().where("first_name", "like", "John%");
```

### Chain includes for readability

```ts
// ✅ Readable
User.query().include("posts").include("profile").include("roles");
```

### Use select() for large datasets

```ts
// ✅ Reduce payload size — pass fields as individual arguments
User.query().select("id", "email").get();

// ❌ Unnecessary data transfer
User.query().get(); // fetches all fields
```

### Destructure get() results

```ts
// ✅ Correct — destructure the tuple
const [users, meta] = await User.query().get();
console.log(users.length);
console.log(meta.total);

// ❌ Incorrect — get() returns a tuple, not an array
const [users] = await User.query().get();
console.log(users.length); // TypeError
```

### Reuse query builders

```ts
const baseQuery = User.query()
  .where("active", "=", true)
  .orderBy("lastname", "asc");

const [admins] = await baseQuery.where("role", "=", "admin").get();
const [users] = await baseQuery.where("role", "=", "user").get();
```
