# Query Builder

Chain filters, sorts, includes, and pagination with a fluent, type-safe API.

## Introduction au Query Builder

The query builder provides a fluent interface for constructing API requests:

```ts
const users = await User.query()
  .where("firstname", "like", "Eli%")
  .where("age", ">", 18)
  .orderBy("lastname", "asc")
  .include("posts")
  .limit(25)
  .get();
```

Every model has a static `query()` method that returns a new query builder instance.

## Requêtes de base

### Récupérer par clé primaire

```ts
const user = await User.query().findByKey(1);
```

Returns a single model instance or throws if not found.

For composite keys:

```ts
const pivot = await UserRole.query().findByKey({ userId: 1, roleId: 5 });
```

### Récupérer plusieurs ressources

```ts
const users = await User.query().get();
```

Returns an array of model instances.

### Récupérer la première ressource

```ts
const user = await User.query().where("email", "=", "john@example.com").first();
```

Returns the first matching instance or `null`.

## Filtres

### Clause where

```ts
const users = await User.query().where("firstname", "like", "John%").get();
```

**Syntax:** `where(field, operator, value)`

### Filtres multiples

Chain multiple `where()` calls:

```ts
const users = await User.query()
  .where("age", ">", 18)
  .where("active", "=", true)
  .where("country", "=", "FR")
  .get();
```

All conditions are combined with `AND`.

### Opérateurs supportés

- **`=`** — equal
- **`!=`** — not equal
- **`>`** — greater than
- **`>=`** — greater than or equal
- **`<`** — less than
- **`<=`** — less than or equal
- **`like`** — pattern matching (use `%` wildcards)
- **`in`** — value in array
- **`not in`** — value not in array

Examples:

```ts
// Like
User.query().where("email", "like", "%@example.com");

// In
User.query().where("status", "in", ["active", "pending"]);

// Not in
User.query().where("role", "not in", ["admin", "moderator"]);
```

## Tri

### Tri simple

```ts
const users = await User.query().orderBy("firstname", "asc").get();
```

**Syntax:** `orderBy(field, direction)`  
**Direction:** `'asc'` or `'desc'`

### Tri multiple

```ts
const users = await User.query()
  .orderBy("lastname", "asc")
  .orderBy("firstname", "asc")
  .get();
```

Sorts are applied in the order they're chained.

## Inclusion de relations

### Include simple

```ts
const user = await User.query().include("posts").findByKey(1);
```

Eager-loads the `posts` relation.

### Include avec contraintes

Apply filters and sorts to the included relation:

```ts
const user = await User.query()
  .include("posts", (q) => q.orderBy("created_at", "desc").limit(5))
  .findByKey(1);
```

The callback receives a query builder for the related model.

### Include imbriqués

Nest includes for deep loading:

```ts
const user = await User.query()
  .include("posts", (q) => q.include("comments", (c) => c.include("author")))
  .findByKey(1);
```

Loads users → posts → comments → authors in one request.

### Include multiples

Chain multiple `include()` calls:

```ts
const user = await User.query()
  .include("posts")
  .include("profile")
  .include("roles")
  .findByKey(1);
```

## Pagination

### Pagination basique

```ts
const users = await User.query().limit(10).offset(20).get();
```

- **`limit(n)`** — number of results
- **`offset(n)`** — skip n results

### Pagination avec métadonnées

```ts
const response = await User.query().limit(25).offset(0).paginate();

console.log(response.data); // model instances
console.log(response.total); // total count
console.log(response.perPage); // items per page
console.log(response.currentPage);
```

## Sélection de champs

Limit which fields are returned:

```ts
const users = await User.query().select(["id", "firstname", "lastname"]).get();
```

**Note:** Primary key fields are always included.

Useful for reducing payload size when you only need specific fields.

## Exemple complet

Combining all features:

```ts
const users = await User.query()
  .where("age", ">=", 18)
  .where("active", "=", true)
  .where("country", "in", ["FR", "BE", "CH"])
  .orderBy("lastname", "asc")
  .orderBy("firstname", "asc")
  .include("posts", (q) =>
    q.where("published", "=", true).orderBy("created_at", "desc").limit(5),
  )
  .include("profile")
  .select(["id", "firstname", "lastname", "email"])
  .limit(25)
  .offset(0)
  .get();
```

This query:

1. Filters users by age, active status, and country
2. Sorts by lastname then firstname
3. Includes published posts (latest 5) and profile
4. Selects specific fields
5. Paginates (25 per page, first page)

## Bonnes pratiques

### Always use typed field names

```ts
// ✅ Type-safe
User.query().where("firstname", "like", "John%");

// ❌ Typo will cause runtime error
User.query().where("first_name", "like", "John%");
```

### Chain includes for readability

```ts
// ✅ Readable
User.query().include("posts").include("profile").include("roles");

// ❌ Less readable
User.query().include("posts").include("profile").include("roles");
```

### Use select() for large datasets

```ts
// ✅ Reduce payload size
User.query().select(["id", "email"]).get();

// ❌ Unnecessary data transfer
User.query().get(); // fetches all fields
```

### Reuse query builders

```ts
const baseQuery = User.query()
  .where("active", "=", true)
  .orderBy("lastname", "asc");

const admins = await baseQuery.where("role", "=", "admin").get();
const users = await baseQuery.where("role", "=", "user").get();
```
